CS 330 – Computational Graphics and Visualization
Portfolio Reflection – 3D Scene Project

This project represents my work in building a fully realized 3D scene using OpenGL, textures, lighting, and camera controls. The scene I created includes a pencil, a metal ball, an eraser, and a wooden ring. These objects were intentionally selected because they combine different materials, shapes, and surface properties that allowed me to demonstrate lighting, texture mapping, transformations, and camera navigation within a single cohesive environment.

The ZIP folder containing the 3D Scene and the Design Decisions document together showcase not only the final result, but also my understanding of the development process required to create and render a 3D world from code.

How do I approach designing software?

Working on this project helped me develop stronger design habits before writing code. Rather than jumping directly into implementation, I began by thinking about what objects I wanted in my scene and what each object would allow me to demonstrate in terms of graphics concepts. For example, the metal ball allowed me to emphasize lighting and material reflection, while the wooden ring and pencil allowed me to focus on textures and transformations.

One new design skill I developed was thinking in terms of renderable components. I learned to break down a visual scene into individual objects, materials, textures, and transformations that could be managed systematically. This helped me understand how complex scenes are really collections of simpler parts working together.

The design process I followed included:

Sketching out the scene conceptually
Identifying required shapes and textures
Planning how the camera should navigate the scene
Determining what custom functions would be needed to manage textures, lighting, and rendering

These tactics are highly transferable to future work. Planning systems visually and logically before implementation is valuable not only in graphics programming but in any large software project.

How do I approach developing programs?

During development, I used several new strategies that were specific to graphics programming. One of the most important was organizing my code to separate scene management, shader management, and texture handling. This separation made debugging much easier when visual issues occurred.

Iteration played a major role in my development process. Very rarely did something work perfectly on the first attempt. I frequently adjusted:

Object scaling and positioning
Lighting values and material properties
Texture bindings and shader parameters
Camera controls and input handling

Through each milestone, my approach to coding became more methodical. Early on, I would make large changes at once and struggle to identify problems. By the end of the project, I was making small, controlled changes and testing constantly, which made development more efficient and less frustrating.

I also became much more comfortable reading and modifying provided framework code, which is an important real-world skill when working with existing codebases.

How can computer science help me in reaching my goals?

This project showed me how computational graphics combine mathematics, programming, and creativity into a practical application. Understanding how 3D transformations, lighting models, and textures work at a low level gives me a deeper understanding of how game engines and visualization software operate.

From an educational standpoint, this knowledge strengthens my foundation in areas such as linear algebra, rendering pipelines, and shader programming, which are critical for advanced graphics, simulation, and game development courses.

From a professional standpoint, this project gave me experience that is directly relevant to careers in:

Game development
Simulation and visualization
Graphics programming
Interactive application development

Being able to demonstrate a working 3D scene built from code shows employers that I understand not only programming concepts but also how those concepts translate into visual, interactive systems.
